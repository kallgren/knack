# Single-blob localStorage shape with versioned migrations

Persisted state lives under a single localStorage key as one JSON document of the form `{:schema-version N, :jobs {...}}`. Each persisted entity is its own top-level collection so today's blob shape mirrors tomorrow's relational schema, table-for-key. On load, the storage adapter reads `:schema-version`, runs any registered migration functions in sequence to reach the current version, then validates the result against the Malli schema before exposing it. Granular per-entity keys were rejected because they lose atomicity and the perf gain is meaningless at this scale. The migration discipline ships in the tracer bullet; retrofitting it after the first shape change would mean recovery code instead of pure functions.
